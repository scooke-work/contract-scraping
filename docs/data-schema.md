# Data Schema Reference

What the scraper writes to `output/` (or any `--output` dir), field by field — so you can
consume the corpus for RAG/embeddings, analytics, or a CLM integration without reverse-
engineering it. The `output/` dir is **gitignored** (regenerated by a run); the formats below
are the contract.

```
output/
├── chunks.jsonl          # RAG payload — one overlapping text chunk per line
├── knowledge_base.md     # human-readable digest, grouped by source
├── agreements.jsonl      # structured fields, one row per real agreement (from extract_structured.py)
├── manifest.json         # run summary
└── raw/<source-slug>/<id>-<title>.json   # one clean record per page/agreement
```

**Regenerate:** `python scrape_contracts.py` (writes chunks/raw/manifest/knowledge_base) →
`python extract_structured.py` (writes agreements.jsonl) → `python scripts/benchmark_edgar.py`
(reads `raw/` → `docs/edgar-benchmark.md`).

**ID conventions:** a record `id` is `sha1(url)[:12]`; a `chunk_id` is `"{record id}-{chunk_index}"`.

---

## `chunks.jsonl` — the RAG payload
One JSON object per line; each is one overlapping, paragraph-aware text chunk (target ~800
words, ~100-word overlap; see `config.chunk_words`/`chunk_overlap` in `sources.yaml`).

| Field | Type | Description | Example |
|---|---|---|---|
| `chunk_id` | string | `{record id}-{chunk_index}` | `"feaa7fd26394-0"` |
| `source` | string | Source display name (from `sources.yaml`) | `"SEC EDGAR Contracts"` |
| `category` | string | Source category label (see [Reference values](#reference-values)) | `"real-agreements"` |
| `url` | string | Canonical URL of the source page/exhibit | `"https://www.sec.gov/Archives/edgar/data/.../d361263dex1011.htm"` |
| `title` | string | Page/agreement title (may be empty) | `"Amended and Restated Reseller Agreement…"` |
| `chunk_index` | int | 0-based position of this chunk within its record | `0` |
| `text` | string | The chunk text | `"AMENDED AND RESTATED RESELLER AGREEMENT…"` |

## `raw/<source-slug>/<id>-<title>.json` — per-page records
One clean record per page (web sources) or per exhibit (EDGAR). `<source-slug>` is a slugified
source name (e.g. `sec-edgar-contracts`, `wikipedia`, `cornell-lii`).

**Common fields (all sources):**

| Field | Type | Description | Example |
|---|---|---|---|
| `id` | string | `sha1(url)[:12]` | `"00586bfb24da"` |
| `source` | string | Source display name | `"SEC EDGAR Contracts"` |
| `category` | string | Source category | `"real-agreements"` |
| `url` | string | Canonical URL | `"https://www.sec.gov/Archives/edgar/data/350563/.../dex1029.htm"` |
| `fetched_at` | string | ISO-8601 UTC fetch timestamp | `"2026-06-21T15:37:30.793457+00:00"` |
| `word_count` | int | Word count of `text` | `3122` |
| `title` | string | Extracted title (may be empty) | `"CONSULTING AGREEMENT/EUSTACE"` |
| `author` | string | Extracted author (often empty) | `""` |
| `date` | string | Extracted publish date (often empty) | `""` |
| `description` | string | Extracted description/meta (often empty) | `""` |
| `text` | string | The clean main text (EDGAR header noise stripped) | `"CONSULTING AGREEMENT… TECO ENERGY, INC. …"` |

**EDGAR-only extra fields** (present when `category == "real-agreements"`):

| Field | Type | Description | Example |
|---|---|---|---|
| `company` | string | Filer display name + CIK | `"TECO ENERGY INC  (CIK 0000350563)"` |
| `form` | string | Exhibit type (filtered to `EX-10*`) | `"EX-10.29"` |
| `file_date` | string | Filing date `YYYY-MM-DD` | `"2004-03-15"` |
| `matched_query` | string | The taxonomy phrase that surfaced it | `"consulting agreement"` |

> The raw record is the **source of truth**; `chunks.jsonl` is derived from `text`, and
> `agreements.jsonl` is derived from the `real-agreements` raw records.

## `agreements.jsonl` — structured fields (optional pass)
One JSON object per real agreement, written by `extract_structured.py` (LLM-backed if
`ANTHROPIC_API_KEY` is set, else heuristic). Carries the raw record's metadata plus extracted
fields. *Not written by the scraper itself — run `python extract_structured.py` to produce it.*

**Carried metadata:** `id`, `source`, `category`, `url`, `company`, `form`, `file_date`,
`matched_query`, `title`, `word_count` (as above).

**Extracted fields:**

| Field | Type | Description |
|---|---|---|
| `agreement_type` | string | e.g. `"Master Services Agreement"` (falls back to `matched_query`) |
| `parties` | string[] | Contracting party names |
| `effective_date` | string | Effective/execution date (`""` if absent) |
| `governing_law` | string | Governing-law jurisdiction, e.g. `"New York"` |
| `term` | string | Initial term / duration |
| `auto_renewal` | string | Renewal mechanics, or `""` |
| `termination` | string | Key termination rights |
| `liability_cap` | string | Limitation-of-liability cap description |
| `indemnification` | boolean | Contains an indemnity? |
| `confidentiality` | boolean | Contains confidentiality obligations? |
| `payment_terms` | string | e.g. `"Net 30"` |
| `clause_inventory` | string[] | Detected clause types (see [taxonomy](#clause_inventory-taxonomy)) |
| `summary` | string | One-line plain-English summary (LLM only) |
| `extraction_method` | string | `"llm"` or `"heuristic"` |

## `manifest.json` — run summary

| Field | Type | Description | Example |
|---|---|---|---|
| `generated_at` | string | ISO-8601 UTC | `"2026-06-21T15:38:46+00:00"` |
| `pages` | int | Total records collected | `370` |
| `chunks` | int | Total chunks written | `4797` |
| `sources` | object | Map of source name → `{pages, words, category}` | `{"SEC EDGAR Contracts": {"pages": 370, "words": 3214853, "category": "real-agreements"}}` |

## `knowledge_base.md`
A single human-readable digest: a header (record/source counts), then each source as a section,
and within it each record as a `### {title}` heading + a source line (`url`, `category`,
`word_count`) + the full `text`. Not machine-structured — use `chunks.jsonl`/`raw/` for that.

---

## Reference values

**`category`** (set per source in `sources.yaml`):

| category | Source(s) |
|---|---|
| `us-law-reference` | Cornell LII (UCC Art. 2, Wex) |
| `concepts` | Wikipedia |
| `international-law` | CISG (Trans-Lex) |
| `business-explainer` | Investopedia *(disabled — 403s automated clients)* |
| `real-agreements` | SEC EDGAR Contracts (EX-10 exhibits) |

**`clause_inventory` taxonomy** — the clause types `extract_structured.py` detects (keyword-
based): `indemnification`, `limitation of liability`, `confidentiality`, `termination`,
`governing law`, `force majeure`, `arbitration`, `assignment`, `warranty`,
`intellectual property`, `auto-renewal`, `payment terms`, `non-solicitation`,
`data protection`, `insurance`. *(The benchmark generator adds `consequential-damages waiver`.)*
For the full negotiated-clause taxonomy and what each means, see the
[clause risk matrix](clause-risk-matrix.md) and [clause bank](clause-library.md).

**`form`** — EDGAR exhibit types are filtered to the `EX-10` ("Material Contracts") family via
`exhibit_types` in `sources.yaml`; values look like `EX-10`, `EX-10.2`, `EX-10.29`.

## How the records relate

```
raw/<source>/<id>.json   (1 clean record: text + metadata)
   │
   ├─►  chunks.jsonl      (N chunks: text split ~800 words, 100 overlap, + metadata)
   │
   └─►  agreements.jsonl  (1 structured row — only for category == real-agreements)
                          via extract_structured.py

manifest.json            rolls up counts across all records/sources
knowledge_base.md        concatenates all record text for human reading
```

One `url` → one raw record (`id`) → many chunks (`chunk_id = {id}-{index}`). Re-runs with
`--append` dedup by `url`. All files are UTF-8; the JSONL files are newline-delimited JSON.
